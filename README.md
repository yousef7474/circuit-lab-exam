# CIRCUIT.LAB — الاختبار القبلي 🔬

نظام اختبار تشخيصي متكامل عبر الإنترنت للطلاب في مسار الإلكترونيات وبرمجة المتحكمات، مع **لوحة معلم حية** تعرض إجابات الطلاب فور تسليمها.

---

## 🎯 الميزات

- ✅ 23 سؤالاً على 5 أقسام، بـ 7 أنواع أسئلة (اختيار، صور، توصيل، صح/خطأ، قراءة كود، نص قصير/مطوّل)
- ✅ **حفظ تلقائي في السحابة** — كل طالب يسلم → تظهر إجاباته في لوحتك مباشرة
- ✅ **لوحة معلم** مع بحث، فرز، تحليل أسئلة، تصدير CSV/JSON
- ✅ مؤقّت 45 دقيقة، شريط تنقل، حفظ محلي احتياطي
- ✅ يعمل على أي جهاز — جوال، تابلت، حاسوب
- ✅ مجاني تماماً (لا رسوم استضافة، لا اشتراكات)

## 📋 خطوات الإعداد الكاملة

### الخطوة 1: إعداد الخادم (Google Sheets — 5 دقائق)

الخادم يخزّن إجابات الطلاب في ورقة Google Sheet تملكها.

#### 1.1 أنشئ ورقة Google Sheet جديدة
- افتح [sheets.google.com](https://sheets.google.com)
- أنشئ ورقة جديدة (فارغة)
- سمِّها مثلاً: `نتائج اختبار قبلي`

#### 1.2 افتح Apps Script
- من قائمة **Extensions → Apps Script**
- ستُفتح صفحة كود جديدة

#### 1.3 الصق الكود
- احذف كل الكود الافتراضي في الصفحة
- افتح ملف `backend.gs` من مجلد المشروع، وانسخ محتواه بالكامل
- الصقه في محرر Apps Script

#### 1.4 حدّد كلمة مرور المعلم
- ابحث عن السطر:
  ```javascript
  const TEACHER_TOKEN = 'CHANGE_ME_TO_A_STRONG_PASSWORD';
  ```
- غيّر النص بين علامتي التنصيص إلى كلمة مرور قوية (12+ حرفاً)
- ⚠️ **احفظ هذه الكلمة** — ستحتاجها للدخول للوحة المعلم

#### 1.5 احفظ ونشر
- اضغط أيقونة الحفظ 💾 (أو `Ctrl+S`)
- اضغط **Deploy → New deployment** (أعلى يمين)
- اضغط أيقونة الترس ⚙️ بجانب "Select type" → اختر **Web app**
- املأ الإعدادات:
  - **Description**: اترك فارغ أو اكتب "Circuit Lab v1"
  - **Execute as**: `Me (your email)`
  - **Who has access**: `Anyone`
- اضغط **Deploy**
- ستُطلب منك الموافقات:
  - `Authorize access` → اختر حساب Google الخاص بك
  - قد تظهر شاشة "Google hasn't verified this app" → اضغط `Advanced → Go to (untitled project)`
  - اضغط `Allow`
- في نهاية العملية، ستحصل على **Web app URL** — انسخه (يبدأ بـ `https://script.google.com/macros/s/...`)

### الخطوة 2: ربط الاختبار بالخادم

- افتح ملف `questions.js` بمحرر نصوص
- ابحث عن هذا القسم في الأعلى:
  ```javascript
  backend: {
    url: ''
  }
  ```
- الصق رابط Web app داخل `url`:
  ```javascript
  backend: {
    url: 'https://script.google.com/macros/s/AKfycb.../exec'
  }
  ```

> 🔒 **مهم:** كلمة المرور (`TEACHER_TOKEN`) موجودة في `backend.gs` **فقط** — لا تضعها في `questions.js` أبداً لأن هذا الملف يصبح عاماً بعد النشر.

### الخطوة 3: نشر الموقع للطلاب

اختر إحدى الطرق التالية:

#### 🟢 الأسرع: Netlify Drop (بدون حساب)
1. افتح [netlify.com/drop](https://app.netlify.com/drop)
2. اسحب **مجلد المشروع بالكامل** داخل الصفحة
3. ستحصل على رابط مثل `stunning-fox-abc123.netlify.app`
4. أرسل الرابط للطلاب

#### 🟢 دائم ومجاني: GitHub Pages (خطوة بخطوة)

**قبل الرفع:** تأكد أنك أنجزت الخطوتين 1 و 2 (Apps Script + رابط في questions.js).

1. **أنشئ حساب GitHub** (إن لم يكن لديك) على [github.com](https://github.com)

2. **أنشئ مستودعاً جديداً:**
   - اضغط `+` → `New repository`
   - Repository name: `circuit-lab-exam` (أو أي اسم إنجليزي)
   - Visibility: **Public** (مطلوب لـ Pages المجاني)
   - لا تُفعّل "Add a README" (لدينا واحد جاهز)
   - اضغط `Create repository`

3. **ارفع الملفات (بدون تنصيب git):**
   - في صفحة المستودع الفارغ، اضغط `uploading an existing file`
   - اسحب هذه الملفات جميعاً من مجلد المشروع:
     - `index.html`
     - `teacher.html`
     - `styles.css`
     - `questions.js` (بعد وضع رابط الخادم فيه)
     - `app.js`
     - `teacher.js`
     - `README.md`
     - `backend.gs` (للأرشيف — لا يُستخدم على الموقع)
   - في الأسفل: اضغط `Commit changes`

4. **فعّل GitHub Pages:**
   - من قائمة المستودع اضغط `Settings`
   - في القائمة الجانبية اضغط `Pages`
   - تحت **Source** اختر `Deploy from a branch`
   - **Branch**: اختر `main` و `/ (root)` ثم اضغط `Save`
   - انتظر 1-2 دقيقة

5. **رابط موقعك:**
   ```
   https://<USERNAME>.github.io/circuit-lab-exam/
   ```
   استبدل `<USERNAME>` باسم مستخدمك في GitHub.

6. **رابط لوحة المعلم:**
   ```
   https://<USERNAME>.github.io/circuit-lab-exam/teacher.html
   ```

7. **جرّب أنت أولاً** قبل مشاركة الرابط مع الطلاب — افتح الاختبار، اكتب اسمك، سلّم، ثم افتح لوحة المعلم وأدخل كلمة المرور. يجب أن ترى نتيجتك.

**لتحديث الأسئلة لاحقاً:**
- اذهب إلى ملف `questions.js` في مستودعك على GitHub
- اضغط أيقونة القلم ✏️ (تعديل)
- عدّل ثم `Commit changes`
- الموقع سيتحدّث تلقائياً خلال دقيقة

#### 🟢 خيار آخر: Vercel أو Cloudflare Pages
كلاهما مجاني ويدعم رفع مجلد الموقع مباشرة.

### الخطوة 4: افتح لوحة المعلم

- افتح: `YOUR-SITE-URL/teacher.html`
- أو للدخول المباشر: `YOUR-SITE-URL/teacher.html?token=YourStrongPassword123`
- أدخل كلمة المرور (نفسها التي في `TEACHER_TOKEN`)
- ستشاهد إجابات الطلاب فور تسليمها 🎉

## ✏️ تعديل الأسئلة

كل الأسئلة موجودة في `questions.js`.

### الأنواع المدعومة:

| النوع | الوصف |
|------|-------|
| `mcq` | اختيار من متعدد |
| `image_mcq` | اختيار من متعدد مع صورة |
| `true_false` | صح / خطأ |
| `matching` | توصيل بين قائمتين |
| `short_text` | إجابة قصيرة (سطر) |
| `long_text` | إجابة مطوّلة (فقرة) |
| `code_reading` | عرض كود + صندوق إجابة |

### مثال إضافة سؤال:

```javascript
{
  id: 'q24',
  type: 'mcq',
  title: 'ما اسم دالة قراءة قيمة تناظرية؟',
  options: [
    { key: 'أ', text: 'digitalRead()' },
    { key: 'ب', text: 'analogRead()' },
    { key: 'ج', text: 'analogWrite()' },
    { key: 'د', text: 'pinMode()' }
  ],
  points: 4
}
```

### إضافة صورة:

```javascript
// من رابط:
image: { url: 'https://example.com/image.png', caption: 'الصورة' }

// SVG مضمّن:
image: { svg: '<svg viewBox="0 0 100 100">...</svg>' }
```

راجع أعلى `questions.js` — يوجد مكتبة SVG جاهزة للاستخدام والتعديل.

## 🎓 لوحة المعلم — ماذا يمكنك فعله

| العملية | الطريقة |
|--------|--------|
| رؤية كل الطلاب المسلمين | القائمة الجانبية (تحديث تلقائي كل 30 ث) |
| البحث عن طالب | حقل البحث بالاسم أو الصف |
| فرز النتائج | القائمة المنسدلة (الأحدث/الاسم/الدرجة) |
| رؤية إجابات طالب واحد | اضغط على اسمه |
| رؤية توزيع الإجابات على كل سؤال | تبويب "تحليل الأسئلة" |
| تحميل ملف واحد لطالب | زر "تحميل" في صفحة تفاصيله |
| تصدير كل النتائج للاكسل | زر "تصدير CSV" |
| تصدير نسخة احتياطية كاملة | زر "تصدير JSON" |
| حذف نتيجة | زر "حذف" (يحذف من الخادم) |

## 🔒 الأمان والخصوصية

- ✅ الرابط الأساسي (`index.html`) عام — يستطيع أي طالب فتحه بدون تسجيل
- ✅ لوحة المعلم محمية بكلمة مرور (`TEACHER_TOKEN`)
- ✅ البيانات محفوظة في Google Sheet تملكها أنت فقط
- ⚠️ **لا تشارك رابط Apps Script مع الطلاب** — استخدم رابط الموقع فقط
- ⚠️ **لا تشارك كلمة `TEACHER_TOKEN`** — احفظها لنفسك
- 💡 يمكنك رؤية كل البيانات مباشرة في الـ Google Sheet أيضاً (بدون لوحة المعلم)

## 🛠️ التخصيص

### تغيير مدة الاختبار
في `questions.js`:
```javascript
durationSeconds: 45 * 60,  // ← غيّر 45 إلى عدد الدقائق
```

### تغيير عدد الأسئلة
احذف أو أضف كائنات في مصفوفة `questions` داخل كل قسم.

### تغيير الألوان
افتح `styles.css` وابحث عن قسم `:root { ... }` — كل الألوان معرّفة كمتغيرات.

## 🐛 مشاكل شائعة

### "تعذّر الاتصال بالخادم" في لوحة المعلم
- تأكد من نشر Apps Script كـ **Web app** بصلاحية "Anyone"
- تأكد من نسخ الرابط الصحيح (يبدأ بـ `https://script.google.com/macros/s/`)
- تأكد من مطابقة `TEACHER_TOKEN` في `backend.gs` وفي `questions.js`

### الطالب يرى "فشل الإرسال"
- الطالب يستطيع دائماً استخدام أزرار التصدير اليدوية (JSON, CSV, بريد)
- تحقق من عدم انتهاء صلاحية نشر Apps Script (نادر جداً)
- إذا عدّلت `backend.gs`، **يجب** إعادة النشر (`Deploy → Manage deployments → Edit → New version`)

### الطالب فقد إجاباته
- الإجابات تُحفَظ محلياً كل 10 ثوانٍ
- إذا أغلق المتصفح، يستطيع العودة إلى نفس الرابط والضغط على "استئناف اختبار سابق محفوظ"

### الخط لا يظهر
- الخطوط تُحمَّل من Google Fonts — يحتاج الطالب اتصال إنترنت
